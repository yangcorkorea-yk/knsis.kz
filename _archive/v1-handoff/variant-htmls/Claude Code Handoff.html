01 How to hand this to Claude Code

Paste the prompt below into Claude Code with the unzipped package as the working directory.
# Drop the unzipped folder at the repo root, then: Read claude-code-handoff/CLAUDE.md end to end. Then read claude-code-handoff/wbs/M0-foundation.md. # For each task in the phase file: 1. Read the task spec (deps, DoD, owner, est). 2. Read referenced docs (e.g. docs/02-data-model.md for schema work). 3. Implement the smallest vertical slice that satisfies DoD. 4. Run pnpm typecheck && pnpm lint && pnpm test && pnpm price:check. 5. Open a PR titled with the task ID (e.g. "M2-03: treatment detail page"). 6. Move to next task. # When a phase's "Phase done when" gate passes, move to the next file. # Stop and ping the PM if you hit an OPEN question (docs/99-open-questions.md). # Hard rules — never break (see CLAUDE.md §2): # - No prices anywhere (CI lint enforces it). # - PII encrypted at rest. 02 Hard rules — never break

Lifted from CLAUDE.md §2. Bake these into PR templates and CI.
01
No prices, ever.
No price field in schema, UI, templates, or content. A CI lint (pnpm price:check) greps source and fails PRs. Quotes are delivered 1-on-1 by a manager.
02
PII encrypted at rest.
Phone, email, photos. Photos live in a private bucket served by 5-min signed URLs only.
03
Consent captured + timestamped.
ToS required. Marketing + photo-publishing consents optional. Schema: boolean + consentedAt column.
04
Audit-log every admin mutation.
Actor, action, entity, before, after, IP, UA, timestamp.
05
i18n catalogs are source-of-truth.
No hardcoded strings in components past M1. pnpm i18n:check blocks PRs on missing or TODO: review keys.
06
No medical claims in UI copy.
Every treatment and clinic page carries the disclaimer.
07
PWA only in v1.
No native apps. Schema is forward-compatible for an Expo wrapper later.

05 Phase task sheets wbs/

Execute top to bottom. Each phase opens with its "Phase done when" gate. ~125 person-days total.
M0 · WEEK 1–2
Foundation
Setup · tokens · base UI · auth scaffold
  • Monorepo + CI/CD + Vercel previews
  • Supabase + Prisma v0
  • Design tokens → Tailwind + Pretendard
  • 15 base components + Storybook playground
  • next-intl scaffold + PWA manifest
Done when: deployable shell with locale routing + 8+ primitives.
M1 · WEEK 2–3
Auth & profile
Phone OTP · consent · read-only profile
  • OTP via Mobizon / Twilio Verify
  • 3-screen login → OTP → consent flow
  • Better-Auth session + middleware
  • RBAC primitives (5 roles)
Done when: user signs in, lands on /me, signs out — all 3 langs.
M2 · WEEK 3–5
Discovery surface
Browsable public site, no lead capture yet
  • Home · Categories · Treatment · Clinics · Reviews
  • Before/After gallery + Search results
  • Seed pipeline (CSV import)
Done when: entire public site browsable in 3 languages.
M3 · WEEK 4–6
Lead capture
Consult form · photo upload · admin handoff
  • Multi-step consult form (Zod)
  • Presigned upload + EXIF strip
  • Lead create + idempotency-key
  • Turnstile + rate limits
Done when: e2e — submit lead, see it in admin inbox.
M4 · WEEK 5–7
KV · chat · notifications
VIP path — parallelizable with M3
  • Korea Visit 3-screen flow
  • In-app chat (Supabase Realtime)
  • Notifications inbox + web push
  • Settings screen wiring
Done when: discover → consult → chat → KV plan works end-to-end.
M5 · WEEK 6–9
Admin core
Desktop manager workbench
  • Admin shell (sidebar + topbar + drawer)
  • Dashboard · Leads · Customers · Clinics
  • Review moderation (with price-keyword scan)
  • Managers + permission matrix
Done when: a manager runs their whole day inside the admin.
M6 · WEEK 8–10
Notifications workflow
Templates · automation · send log · B/A
  • Template editor (4 channels × 3 langs)
  • Automation rules + Inngest workers
  • WhatsApp / Telegram / SMS adapters
  • Send log with retry · B/A admin
Done when: "lead received → templated WA out in 5 min" runs hands-off.
M7 · WEEK 10–12
Hardening & launch
Perf · a11y · i18n QA · security · soft launch · GA
  • Lighthouse budget in CI
  • WCAG 2.1 AA sweep
  • Native KZ + KR review pass
  • 5 Playwright journeys
  • Soft launch (50 users / 7 days) → GA
Done when: DNS cut to knsis.kz, status page live, runbook committed.
